Introduction à l'API

L'API d'une application Nextcloud est développé en PHP, vu que Nextcloud est développé sur framework Symfony.

Dans cette introduction je vais présenter rapidement comment créer rapidement une nouvelle route d'API.

Création d'une nouvelle route​

Chaque route de l'API renvoie vers une fonction inscrite dans les Controllers de l'application.

Par exemple pour redigirer les utilisateurs vers la page de garde de l'application, il faut modifier:

  • le fichier de route appinfo/routes.php
  • le fichier controller : lib/Controller/PageController.php
  • le template vers lequel sera diriger l'utilisateur : templates/index.php

Ajout d'une route​

appinfo/routes.php
<?php
return [
    'routes' => [
        ['name' => 'page#index', 'url' => '/', 'verb' => 'GET'],
        [
            'name' => 'page#functionName',
            'url' => '/apilink',
            'verb' => 'GET'
        ],
    ],
];

le parametre name d'une route se compose toujours de la première partie du nom du controller. Par exemple ici le fichier controller se nomme PageController.php, donc la route sera page#nomdelafonction.

La route sera ensuite accessible via l'adresse nomdedomaine.fr/index.php/apps/appname/route.

Ajout d'une fonction dans le Controller​

lib/Controller/PageController.php
<?php

namespace OCA\IntranetAgglo\Controller;

use OCP\IRequest;
use OCP\AppFramework\Http\TemplateResponse;
use OCP\AppFramework\Controller;
use OCA\IntranetAgglo\AppInfo\Application;
use OCP\AppFramework\Http\DataResponse;
use OCP\IGroupManager;
use OCP\IUserSession;
use OCP\IUser;


class PageController extends Controller
{
    public function __construct(IRequest $request, IGroupManager $groupmanager, IUserSession $session)
    {
        parent::__construct(Application::APP_ID, $request);
        $this->groupmanager = $groupmanager;
        $this->session = $session;
    }

    /**
     * @NoAdminRequired
     * @NoCSRFRequired
     */
    public function index()
    {
        return new TemplateResponse(Application::APP_ID, 'index');  // templates/index.php
    }
}

ici la fonction index redirige vers un Template , à savoir index.php.

Toutes les librairies contenant les interfaces et méthodes de Nextcloud sont répertorié dans leur documentation développeur. Elle est disponible sur ce site https://nextcloud-server.netlify.app/

Création d'un template​

Le template est le front de l'application.

Les fonctions de php pour pouvoir ajouter du css et du javascript :

templates/index.php
style('nomdelapp', 'nomdufichierstyle');
script('nomdelapp', 'nomdufichierscript');

le nom des fichier est sans l'extension .css ou .js.

S'il y a besoin d'intégrer plusieurs fichiers la string nomdufichier peut également être remplacé par un tableau de String.

Ajouter une Base de données​

Créer une BDD​

La création et modification d'une table de données de la bdd de Nextcloud se fait via les fichiers de lib/Migration/.

Le nom du fichier doit être formaté de manière suivant : Version000000Date20220628120000.

Exemple de fichier de migration:

lib/Migration/Version000000Date20220628120000.php
<?php

namespace OCA\AppName\Migration;

use Closure;
use OCP\DB\ISchemaWrapper;
use OCP\Migration\SimpleMigrationStep;
use OCP\Migration\IOutput;

class Version000000Date20220628120000 extends SimpleMigrationStep
{

    /**
     * @param IOutput $output
     * @param Closure $schemaClosure The `\Closure` returns a `ISchemaWrapper`
     * @param array $options
     * @return null|ISchemaWrapper
     */
    public function changeSchema(IOutput $output, Closure $schemaClosure, array $options)
    {
        /** @var ISchemaWrapper $schema */
        $schema = $schemaClosure();

        if (!$schema->hasTable('AppnameTablename')) {
            $table = $schema->createTable('AppnameTablename');
            $table->addColumn('id', 'integer', [
                'autoincrement' => true,
                'notnull' => true,
            ]);
            $table->addColumn('author', 'string', [
                'length' => 200,
            ]);
            $table->addColumn('title', 'text', [
                'default' => ''
            ]);
            $table->addColumn('boolvar', 'boolean', [
                'default' => false,
                'notnull' => false,
            ]);
            $table->setPrimaryKey(['id']);
        }
        return $schema;
    }
}

Lire les entrées d'une bdd​

on peut créer une Entité object de la bdd grâce un fichier de ce style dans le répertoire lib/Db/:

lib/Db/Object.php
<?php

namespace OCA\AppName\Db;

use JsonSerializable;

use OCP\AppFramework\Db\Entity;

class Object extends Entity implements JsonSerializable //Object est le nom de votre objet
{

    protected $author;
    protected $title;
    protected $boolvar;

    public function jsonSerialize()
    {
        return [
            'id' => $this->id,
            'author' => $this->author,
            'title' => $this->title,
            'boolvar' => $this->boolvar
        ];
    }
}

lib/Db/ObjectMapper.php
<?php

namespace OCA\AppName\Db;

use OCP\AppFramework\Db\DoesNotExistException;
use OCP\AppFramework\Db\Entity;
use OCP\AppFramework\Db\QBMapper;
use OCP\DB\QueryBuilder\IQueryBuilder;
use OCP\IDBConnection;

class ObjectMapper extends QBMapper //Object est le nom de votre objet
{

    public function __construct(IDBConnection $db)
    {
        parent::__construct($db, 'AppnameTablename', Object::class); //Object est le nom de votre objet
    }

    /**
     * @param int $id
     * @return Entity|Apps
     * @throws \OCP\AppFramework\Db\MultipleObjectsReturnedException
     * @throws DoesNotExistException
     */
    public function find(int $id): Object //Object est le nom de votre objet
    {
        /* @var $qb IQueryBuilder */
        $qb = $this->db->getQueryBuilder();

        $qb->select('*')
            ->from($this->getTableName())
            ->where(
                $qb->expr()->eq('id', $qb->createNamedParameter($id))
            );

        return $this->findEntity($qb);
    }

    /**
     * @return array
     */
    public function findAll(): array
    {
        /* @var $qb IQueryBuilder */
        $qb = $this->db->getQueryBuilder();

        $qb->select('*')
            ->from($this->getTableName());

        return $this->findEntities($qb);
    }
}

La Base de données de Nextcloud est gérée via l'utilisation de Doctrine , pour plus d'informations voir la doc de Doctrine de Symfony.

Ajout des Notifications​

Pour ajouter l'utilisation des notifications de Nextcloud il faut modifier 3 fichiers:

  • Un controller qui gerera la création de la notification
  • Application.php
  • Notifier.php

Création de Notifier.php​

Ce fichier sera situé dans le répertoire lib/Notification/ et se présentera sous la forme suivante :

lib/Notification/Notifier.php
<?php

declare(strict_types=1);

namespace OCA\AppName\Notification;

use OCA\AppName\AppInfo\Application;

use InvalidArgumentException;
use OCP\IURLGenerator;
use OCP\Notification\INotification;
use OCP\Notification\INotifier;
use OCP\Notification\AlreadyProcessedException;
use OCP\Notification\IManager as INotificationManager;

use OCP\IUserManager;
use OCP\IUser;

class Notifier implements INotifier
{
    /** @var IURLGenerator */
    private $urlGenerator;

    public function __construct(IURLGenerator $urlGenerator,  INotificationManager $notificationManager, IUserManager $userManager)
    {
        $this->urlGenerator = $urlGenerator;
        $this->notificationManager = $notificationManager;
        $this->userManager = $userManager;
    }

    /**
     * Identifier of the notifier
     * @return string
     */
    public function getID(): string
    {
        return Application::APP_ID;
    }

    /**
     * Human readable name describing the notifier
     */
    public function getName(): string
    {
        return 'Application Exemple';
    }

    /**
     * @param INotification $notification
     * @param string $languageCode The code of the language that should be used to prepare the notification
     */
    public function prepare(INotification $notification, string $languageCode): INotification
    {
        if ($notification->getApp() !== Application::APP_ID) {
            throw new \InvalidArgumentException();
        }

        $notification->setParsedSubject($notification->getSubject())
            ->setRichSubject('{app} - ' . $notification->getSubject(), [
                'app' => [
                    'type' => 'app',
                    'id' => $notification->getObjectType(),
                    'name' => $notification->getApp(),
                ]
            ]);
        $notification->setParsedMessage($notification->getMessage());

        $notification->setLink($this->urlGenerator->linkToRouteAbsolute('appname.page.index') . '#' . $notification->getObjectId());

        $notification->setIcon($this->urlGenerator->getAbsoluteURL($this->urlGenerator->imagePath('intranetagglo', 'LogoCA2B.Mpng')));

        return $notification;
    }
}

pour voir toutes les options possible concernant la modification de la notification dans la fonction prepare , se référencer à la doc de Nextcloud.

Déclaration du Service de Notification (Et du Widget)​

lib/AppInfo/Application.php
<?php

namespace OCA\AppName\AppInfo;


use OCA\AppName\Notification\Notifier;
use OCA\AppName\Dashboard\Widget;

use OCP\Notification\IManager;
use OCP\AppFramework\App;
use OCP\AppFramework\Bootstrap\IBootContext;
use OCP\AppFramework\Bootstrap\IBootstrap;
use OCP\AppFramework\Bootstrap\IRegistrationContext;

class Application extends App implements IBootstrap
{
    public const APP_ID = 'appname';

    public function __construct()
    {
        parent::__construct(self::APP_ID);
    }

    public function register(IRegistrationContext $context): void
    {
        $context->registerDashboardWidget(Widget::class);
    }

    public function boot(IBootContext $context): void
    {
        $manager = $context->getAppContainer()->query(IManager::class);
        $manager->registerNotifierService(Notifier::class);
    }
}

Supprimer une notification​

Pour supprimer une notification il faut recréer un object de classe notification , et lui assigner les valeurs qui permettent de l'identifier pour la suppression.

lib/Controller/ObjectController.php
$notification = $this->NotificationManager->createNotification();
$notification->setApp(Application::APP_ID)
  ->setObject('news', $id);
$this->NotificationManager->markProcessed($notification);
